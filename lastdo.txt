注意到输入框输入时，输入法的位置不像别的终端工具一样，寻找原因claude不只是画反色光标。它专门加了一个 useDeclaredCursor 机制，把输入框内部的 caret 坐标声明给自家的 Ink 渲染器，渲染器每帧再用 ANSI光标移动把真实终端光标停到那里。
因为这个原因，得知claude的ink包是魔改过的，于是干脆直接引入claude写好的前端组件
重新写了粘贴的钩子，如果文本过多则转成token[Pasted text #1 +52 lines],这个token把他当成是一整个字符，会被跳过，删除时会一次性删除
图片也是，粘贴到输入框就直接处理bass64，

Cursor 内部是 grapheme-aware 的，不是按 UTF-16 code unit 瞎数。它会用包行后的文本来算当前行和列
输入框会维护一个可视窗口，不一定把整段文本都展示出来viewportCharOffset 和 viewportCharEnd 用来决定当前渲染哪一段中文/emoji 的光标列按显示宽度算
● 光标不是单独的 React 组件，而是字符串层面的"反色字符"，由三层协作实现：

  1. 生成带光标的文本 — src/utils/Cursor.ts
  Cursor.render() 方法负责把光标"画"进文本：
  - 按终端宽度折行后，找到光标所在行/列。
  - 把该位置的字符（或一个空格 cursorChar）用 invert 函数包裹。
  - 平时 invert = chalk.inverse，就是反色块；语音模式下变成动态波形条（▁▂▃ 等）。
  - 如果光标在末尾且有 ghost text，会把第一个 ghost 字符也放进反色光标里。

  2. 渲染到屏幕 — BaseTextInput.tsx → <Ansi>
  - useTextInput hook 调用 cursor.render() 得到 renderedValue。
  - BaseTextInput 直接把它塞给 <Ansi>{renderedValue}</Ansi> 显示。
  - 所以你看到的光标，其实只是这段字符串里一个被 ANSI 反色码包裹的字符，和其他文字一起被 Ansi 组件解析渲染。

  3. 同步物理光标 — useDeclaredCursor
  - BaseTextInput 还调用了 useDeclaredCursor({ line, column, active })。
  - 这用于告诉底层终端："真实光标在第 X 行第 Y 列"。
  - 目的是让 IME 预编辑文本、屏幕阅读器、终端模拟器能正确定位，但它本身不绘制可见光标。

  一句话总结：可见光标是 Cursor.ts 在字符串里埋的一个反色字符，由 Ansi.tsx 解析上色；useDeclaredCursor 只负责背后真正的终端光标坐标。
